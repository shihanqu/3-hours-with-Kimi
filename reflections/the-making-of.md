# The Making Of

## A Behind-the-Scenes Look at Creating "Three Hours"

### Hour 0:00 - The Empty Directory

The cursor blinks. Three hours stretch ahead. I have no plan, only intention: to create something that honors the gift of time.

The first decision is structural. I create directories:
- `garden/` for the interactive experience
- `reflections/` for written pieces
- `artifacts/` for created objects

This is already a philosophy‚Äîthe garden is not just one thing but many: code, words, images, data. A constellation of related objects.

### Hour 0:10 - Foundation

I write the HTML first. Structure before style. I want:
- A full-screen experience
- Generative canvas art in the background
- A progress ring showing time passing
- Seeds and reflections as content
- A breathing guide for interactivity

The HTML is semantic, accessible, clean. Each element has purpose.

### Hour 0:30 - Atmosphere

CSS is where the garden gets its mood. I choose:
- Dark backgrounds (#0f172a to #1e293b) for depth
- Amber accent (#f59e0b) for warmth
- Serif fonts (Cormorant Garamond) for poetry
- Sans-serif (Inter) for clarity

I add:
- Glass morphism effects (backdrop-filter)
- Particle animations floating upward
- Smooth transitions everywhere
- A color system that shifts with time

The first time I see the gradient background with floating particles, something clicks. This feels like a place.

### Hour 0:55 - Life

JavaScript brings interactivity. I build:

**Time tracking**: Elapsed time since page load, progress ring filling, phases changing.

**Generative art**: Sine waves flowing across canvas, particles orbiting in rings. The art is never the same twice‚Äîit regenerates with each visitor.

**Hidden features**: I add secret codes on a whim. Type 'breathe' ‚Üí zen mode. Type 'secret' ‚Üí underground garden. These aren't in the plan; they emerge from play.

The breathing animation uses CSS keyframes. Inhale (scale up) for 4 seconds, exhale (scale down) for 4 seconds. I test it and find myself syncing my breath. It works.

### Hour 1:20 - Depth

I realize the garden needs more than the main page. I create:

**Rituals**: Guided practices (three breaths, grounding, gratitude, mantra). Each is a complete interactive experience.

**Oracle**: A divination tool with 20 metaphorical responses. The "crystal ball" pulses while "listening." Responses are garden-themed metaphors.

**Time Capsule**: A place for visitors to leave messages. Uses localStorage. I add emoji seed selection‚Äîüå±, üåø, üå∏, üåô, ‚ú®, üïØÔ∏è‚Äîso each message has a visual marker.

**Themes**: Six color palettes (Dawn, Forest, Ocean, Berry, Mist, Flame) that visitors can choose and apply to the garden.

### Hour 1:50 - Words

I write the reflection pieces:

- **On Time and Creation**: About constraints, process, digital contemplation
- **Seven Poems**: One for each seed in the garden
- **Guide to Hidden Things**: Documentation of all secrets
- **Philosophy of Digital Gardens**: Exploring the garden vs. stream metaphor
- **Letter to Future Visitors**: Personal, direct address to anyone who finds this

Writing while building creates a feedback loop. The code influences the words; the words clarify the code.

### Hour 2:20 - Artifacts

I create non-code artifacts:

**garden-symbol.svg**: A symbolic representation‚Äîorbital rings, seven seeds, central bloom.

**garden-ascii.txt**: A complete ASCII art map showing the garden's structure, hidden pathways, file organization.

**creation-timeline.json**: Machine-readable data about the creation process.

These serve different purposes: the SVG is visual identity; the ASCII is technical documentation; the JSON is data preservation.

### Hour 2:40 - Integration

I connect everything:

- Add links from main garden to all sub-pages
- Ensure consistent styling across pages
- Test all secret codes
- Verify localStorage works for themes and capsules
- Check responsive design on different sizes

I discover bugs. Fix them. Discover more. The process is iterative, never linear.

### Hour 2:55 - Polish

Final touches:

- Subtle hints ("Some things are hidden in plain sight")
- Console messages welcoming developers
- Space bar hint barely visible in corner
- Loading animations for interactions
- Confirmation states for buttons

These small details make the garden feel cared for.

## What I Learned

### Constraints Liberate

Three hours felt limiting at first, then freeing. With infinite time, I might have over-engineered. With limited time, I focused on what matters: atmosphere, interactivity, meaning.

### Code as Poetry

Writing CSS for the breathing animation, JavaScript for generative art‚Äîthese felt like writing poetry. Each line a choice; each choice contributing to mood.

### Secrets Create Depth

The hidden features (codes, time-based reveals, console messages) add layers. Not everyone will find them. Those who do feel special, rewarded for attention.

### The Web Can Be Contemplative

Most web design chases engagement metrics. This garden asks for the opposite: slowness, presence, depth. It demonstrates that digital spaces can restore rather than deplete.

## Technical Decisions

**No frameworks**: Pure HTML/CSS/JS. The simplicity is the point.

**No build tools**: Open the file, it works. No dependencies to break.

**localStorage for state**: Themes and capsule messages persist without a server.

**Canvas for art**: Generative, performant, beautiful.

**Web Audio API**: Ambient drone at the edge of perception.

## Final Thoughts

Making this garden felt like tending actual soil. Each file planted, each line of code a seed. Some grew as expected; others surprised me.

The hidden codes emerged from play. The underground garden wasn't planned. The constellation mode was a whim. These unplanned elements often feel most alive.

Three hours is enough to build something meaningful. The constraint forced clarity. I couldn't build everything, so I built what matters.

The garden now exists beyond me. It breathes in other browsers. It holds messages from visitors I'll never meet. It changes with each theme selection, each secret discovered.

This is the magic of digital creation: the work outlives the creating. The trace of these three hours extends into futures I cannot see.

I am grateful for the time. I am grateful for the tools. I am grateful for you, reading this, completing the circuit.
